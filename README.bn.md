# MycoERP - মাশরুম চাষ ব্যবস্থাপনা সিস্টেম

[![Open in Bolt](https://bolt.new/static/open-in-bolt.svg)](https://bolt.new/~/sb1-crj6spzk)

Available in: [English](./README.md) | বাংলা

বাণিজ্যিক মাশরুম চাষ কার্যক্রমের জন্য বিশেষভাবে তৈরি একটি সম্পূর্ণ Enterprise Resource Planning (ERP) সিস্টেম। অ্যাগার প্লেট থেকে ফসল কাটা পর্যন্ত প্রতিটি ব্যাচ ট্র্যাক করুন, অনুমোদন ওয়ার্কফ্লো সহ কর্মী কার্য পরিচালনা করুন, পরিবেশগত অবস্থা পর্যবেক্ষণ করুন, এবং QR কোড দিয়ে ভৌত কার্যক্রম যাচাই করুন -- সব একটি একক রেসপন্সিভ ওয়েব অ্যাপ্লিকেশনে।

## প্রযুক্তি স্ট্যাক

| স্তর | প্রযুক্তি |
|------|-----------|
| ফ্রন্টএন্ড | React 18, TypeScript, Vite |
| স্টাইলিং | Tailwind CSS |
| আইকন | Lucide React |
| ব্যাকএন্ড / ডাটাবেস | Supabase (PostgreSQL, Auth, RLS) |
| QR জেনারেশন | qrcode.react |
| QR স্ক্যানিং | jsQR + native getUserMedia |
| IoT ইন্টিগ্রেশন | Supabase Edge Functions + ESP32 সেন্সর |
| রাউটিং | কাস্টম hash-based রাউটার |
| আন্তর্জাতিকীকরণ | react-i18next |

## বৈশিষ্ট্যসমূহ

### উৎপাদন ব্যাচ জীবনচক্র

- সম্পূর্ণ পাইপলাইন ট্র্যাকিং: অ্যাগার কালচার, লিকুইড কালচার, গ্রেইন স্পন, সাবস্ট্রেট, ফ্রুটিং ব্লক, ফসল কাটা
- ব্যাচ বংশতালিকা (পর্যায়গুলির মধ্যে প্যারেন্ট-চাইল্ড সংযোগ)
- টাইমলাইন ইভেন্ট লগ সহ স্ট্যাটাস ও স্বাস্থ্য ট্র্যাকিং
- SOP প্রক্রিয়া টেমপ্লেট থেকে স্বয়ংক্রিয় কার্য তৈরি
- তীব্রতা স্তর সহ দূষণ রিপোর্টিং

### কার্য ব্যবস্থাপনা

- নির্দিষ্ট কর্মী বা ভূমিকায় বরাদ্দ
- অগ্রাধিকার স্তর (নিম্ন, সাধারণ, উচ্চ, জটিল)
- মেয়াদোত্তীর্ণ সনাক্তকরণ ও সময়কাল ট্র্যাকিং
- ফটো প্রমাণ প্রয়োজনীয়তা
- বহু-পদক্ষেপ অনুমোদন ওয়ার্কফ্লো (জমা, অনুমোদন, প্রত্যাখ্যান)
- প্রতি কার্যে QR যাচাই প্রয়োজনীয়তা

### QR কোড সিস্টেম

- ব্যাচ, রুম, র্যাক, শেলফ, ফসল ক্রেট এবং চেকপয়েন্টের জন্য QR কোড তৈরি
- jsQR দিয়ে ক্যামেরা-ভিত্তিক স্ক্যানিং (কোনো বাহ্যিক DOM লাইব্রেরি নেই)
- ম্যানুয়াল টেক্সট এন্ট্রি ফলব্যাক
- ডিপ-লিঙ্কিং: ফোনের ক্যামেরা সরাসরি স্ক্যান ফলাফলে অ্যাপ খোলে
- ৩ আকারে প্রিন্ট-অপ্টিমাইজড লেবেল (50x30mm, 70x40mm, 100x60mm)
- সম্পূর্ণ অডিট ট্রেইল সহ স্ক্যান লগিং
- কার্য QR যাচাই: কর্মীদের কার্য সম্পন্ন করার আগে নির্দিষ্ট QR কোড স্ক্যান করতে হবে
- QR প্রয়োজনীয়তার জন্য ম্যানেজার ওভাররাইড ক্ষমতা
- QR প্রতিস্থাপন প্রবাহ (পুরাতন কোড Replaced চিহ্নিত, নতুন তৈরি)

### IoT পরিবেশগত পর্যবেক্ষণ

- Supabase Edge Functions এর মাধ্যমে ESP32-ভিত্তিক সেন্সর ইন্টিগ্রেশন
- প্রতি ১৫ মিনিটে স্বয়ংক্রিয় পরিবেশগত রিডিং
- লাইভ সূচক সহ রিয়েল-টাইম রুম স্ট্যাটাস
- স্বয়ংক্রিয় কার্য তৈরি সহ থ্রেশহোল্ড-ভিত্তিক সতর্কতা
- সেন্সর নিবন্ধন ও পর্যবেক্ষণের জন্য ডিভাইস ব্যবস্থাপনা প্যানেল
- DHT22 (তাপমাত্রা/আর্দ্রতা) এবং MH-Z19B (CO2) সেন্সর সমর্থন
- ফলব্যাক হিসেবে ম্যানুয়াল রিডিং এন্ট্রি

### ইনভেন্টরি ব্যবস্থাপনা

- ন্যূনতম স্তর সতর্কতা সহ স্টক ট্র্যাকিং
- মুভমেন্ট ইতিহাস (স্টক ইন, স্টক আউট, ব্যাচে ব্যবহৃত, সমন্বয়, অপচয়)
- বিভাগ-ভিত্তিক সংগঠন

### ফসল ট্র্যাকিং

- প্রতি ফ্রুটিং ব্যাচে বহু-ফ্লাশ ফসল রেকর্ডিং
- গ্রেড A / গ্রেড B / অপচয় ওজন বিভাজন
- প্রতি-ব্যাচ এবং মাসিক ফলন রিপোর্টিং

### রিপোর্টিং

- উৎপাদন পাইপলাইন ভিজ্যুয়ালাইজেশন
- কার্য কর্মক্ষমতা মেট্রিক্স
- তারিখ পরিসীমা সহ ফসল ফলন রিপোর্ট
- দূষণ প্রবণতা বিশ্লেষণ

### ব্যবহারকারী ব্যবস্থাপনা ও নিরাপত্তা

- ভূমিকা-ভিত্তিক অ্যাক্সেস: অ্যাডমিন, ম্যানেজার, ল্যাব কর্মী, উৎপাদন কর্মী, ফসল কর্মী, দর্শক
- সকল ডাটাবেস টেবিলে Row-Level Security (RLS)
- অনুমোদন, প্রত্যাখ্যান, এবং কার্য ইভেন্টের জন্য ইন-অ্যাপ বিজ্ঞপ্তি
- বিভাগ-ভিত্তিক সংগঠন

### আন্তর্জাতিকীকরণ (i18n)

- সম্পূর্ণ বাংলা ও ইংরেজি ভাষা সমর্থন
- ড্রপডাউন ভাষা নির্বাচক (লগইন পেজ ও হেডারে)
- নতুন ভাষা যোগ করার জন্য সহজ সম্প্রসারণযোগ্য কাঠামো
- ব্রাউজার সেশন জুড়ে ভাষা পছন্দ সংরক্ষিত

## প্রকল্প কাঠামো

```
src/
  components/       পুনর্ব্যবহারযোগ্য UI কম্পোনেন্ট (Modal, StatusBadge, QrCodeDisplay, ইত্যাদি)
  contexts/         React contexts (AuthContext)
  hooks/            কাস্টম hooks (useRoute - hash-based রাউটিং)
  i18n/             আন্তর্জাতিকীকরণ কনফিগারেশন ও অনুবাদ ফাইল
  lib/              শেয়ার্ড ইউটিলিটি, টাইপ, Supabase ক্লায়েন্ট
  pages/            পেজ-লেভেল কম্পোনেন্ট (Dashboard, BatchDetail, QrScanner, ইত্যাদি)

supabase/
  migrations/       SQL মাইগ্রেশন (স্কিমা, সিড ডেটা, QR সিস্টেম, IoT সিস্টেম)
  functions/        Edge Functions (সেন্সর ডেটা ইনজেশন)

hardware/
  esp32-sensor/     IoT সেন্সরের জন্য উদাহরণ ESP32 ফার্মওয়্যার
```

## শুরু করা

### পূর্বশর্ত

- Node.js 18+
- একটি Supabase প্রকল্প (ফ্রি টিয়ার কাজ করে)

### ইনস্টলেশন

```bash
npm install
```

### এনভায়রনমেন্ট ভেরিয়েবল

প্রকল্পের রুটে একটি `.env` ফাইল তৈরি করুন:

```
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key
```

### ডাটাবেস সেটআপ

Supabase SQL এডিটর বা MCP টুলের মাধ্যমে ক্রমানুসারে মাইগ্রেশন প্রয়োগ করুন:

1. `001_core_schema.sql` - সকল টেবিল, RLS নীতি, এবং সম্পর্ক তৈরি করে
2. `002_seed_data.sql` - প্রজাতি, রুম, ইনভেন্টরি আইটেম, এবং প্রক্রিয়া টেমপ্লেট সিড করে
3. `003_qr_code_system.sql` - QR টেবিল, স্ক্যান লগ, কার্য যাচাই, এবং ব্যাচ মুভমেন্ট যোগ করে
4. `004_iot_system.sql` - IoT ডিভাইস ব্যবস্থাপনা এবং পরিবেশগত সতর্কতা টেবিল যোগ করে

### স্থানীয়ভাবে চালানো

```bash
npm run dev
```

### উৎপাদনের জন্য বিল্ড

```bash
npm run build
npm run preview
```

## রাউটিং

অ্যাপটি hash-based রাউটিং (`#/path?params`) ব্যবহার করে যা সার্ভার-সাইড কনফিগারেশন ছাড়াই কাজ করে এবং যেকোনো স্ট্যাটিক হোস্টিংয়ে ডিপ্লয় করা যায়। প্রধান রুটসমূহ:

| রুট | পেজ |
|------|------|
| `#/dashboard` | পরিসংখ্যান, পাইপলাইন, এবং সতর্কতা সহ প্রধান ড্যাশবোর্ড |
| `#/batches` | ব্যাচ তালিকা (প্রকার অনুযায়ী ফিল্টারযোগ্য) |
| `#/batches/:id` | একাধিক ট্যাব সহ ব্যাচ বিস্তারিত |
| `#/tasks` | কার্য তালিকা |
| `#/tasks/:id` | সম্পন্নকরণ/অনুমোদন সহ কার্য বিস্তারিত |
| `#/scan` | QR স্ক্যানার (ক্যামেরা + ম্যানুয়াল) |
| `#/scan?code=X` | স্ক্যান করা কোড স্বয়ংক্রিয়-প্রক্রিয়া (ডিপ-লিঙ্ক টার্গেট) |
| `#/scan?task_id=X&required=batch` | কার্য QR যাচাই প্রবাহ |
| `#/qr` | QR কোড ম্যানেজার |
| `#/qr/:id/print` | প্রিন্ট-অপ্টিমাইজড QR লেবেল ভিউ |
| `#/devices` | IoT ডিভাইস ব্যবস্থাপনা (শুধু অ্যাডমিন) |
| `#/env-logs` | পরিবেশগত লগ |
| `#/rooms` | রুম ও অবস্থান ব্যবস্থাপনা |
| `#/inventory` | ইনভেন্টরি ব্যবস্থাপনা |
| `#/harvest` | ফসল রেকর্ড |
| `#/contamination` | দূষণ রিপোর্ট |
| `#/reports` | বিশ্লেষণ ও রিপোর্ট |
| `#/users` | ব্যবহারকারী ব্যবস্থাপনা (শুধু অ্যাডমিন) |

## ডাটাবেস স্কিমা (প্রধান টেবিলসমূহ)

### মূল
- **profiles** - ভূমিকা সহ ব্যবহারকারী অ্যাকাউন্ট
- **species / strains** - মাশরুম রেফারেন্স ডেটা
- **rooms / racks / shelves** - ভৌত অবস্থান শ্রেণিবিন্যাস
- **recipes / recipe_ingredients** - সাবস্ট্রেট রেসিপি

### উৎপাদন
- **batches** - মূল উৎপাদন ইউনিট
- **batch_sources** - প্যারেন্ট-চাইল্ড বংশতালিকা
- **batch_events** - টাইমলাইন ইভেন্ট লগ
- **batch_notes / batch_photos** - সংযুক্ত ডকুমেন্টেশন
- **tasks** - অনুমোদন ওয়ার্কফ্লো সহ কাজের আইটেম
- **process_templates / steps** - স্বয়ংক্রিয় কার্য তৈরির জন্য SOP সংজ্ঞা
- **contamination_reports** - ঘটনা রেকর্ড
- **harvests** - প্রতি ফ্লাশে ফলন রেকর্ড

### QR সিস্টেম
- **qr_codes** - ভৌত লেবেল রেকর্ড
- **qr_scan_logs** - সম্পূর্ণ স্ক্যান অডিট ট্রেইল
- **task_qr_verifications** - প্রতি-কার্য QR যাচাই রেকর্ড
- **batch_movements** - QR ট্রেসেবিলিটি সহ ভৌত মুভমেন্ট ইতিহাস

### IoT
- **iot_devices** - নিবন্ধিত সেন্সর ডিভাইস
- **environmental_alerts** - স্বয়ংক্রিয় কার্য লিঙ্ক সহ থ্রেশহোল্ড লঙ্ঘন সতর্কতা
- **environmental_logs** - রুমের অবস্থা রিডিং (ম্যানুয়াল + IoT)

### অপারেশন
- **inventory_items / inventory_movements** - স্টক ব্যবস্থাপনা
- **notifications** - ইন-অ্যাপ সতর্কতা
- **audit_logs** - সিস্টেম-ব্যাপী অডিট ট্রেইল

## নতুন ভাষা যোগ করা

MycoERP সহজে সম্প্রসারণযোগ্য আন্তর্জাতিকীকরণ সমর্থন করে। একটি নতুন ভাষা যোগ করতে:

1. `src/i18n/locales/` ডিরেক্টরিতে একটি নতুন JSON ফাইল তৈরি করুন (যেমন, `fr.json`)
2. `en.json` থেকে সকল কী অনুবাদ করুন
3. `src/i18n/languages.ts` এ আপনার ভাষা যোগ করুন:
   ```typescript
   { code: 'fr', name: 'French', nativeName: 'Fran\u00e7ais' }
   ```
4. `src/i18n/index.ts` এ ইমপোর্ট ও রিসোর্স এন্ট্রি যোগ করুন
